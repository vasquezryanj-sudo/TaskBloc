I want to build a personal reading tracker web app. Here's what it needs to do:
Scheduling

I can add books with a title, current page, and total pages
The app schedules reading across 3 sessions per week (adjustable), calculating how many pages per session and projecting a finish date
I can add high-priority items (like articles) that displace the next scheduled session(s), pushing everything else forward
Sessions and page targets are editable — if I read more or less than planned, I can update it and the schedule adapts
Sessions should be exportable to Google Calendar (as an ICS file)

Notes

Each book has its own notes section with three tabs: Outline, Scratch, and Quotes
The Quotes tab has fields for the quote text and page number
Notes persist across sessions for that book
I can export a book's notes as a PDF with each tab separated by a divider

Tech

Use Vercel for hosting and Supabase for the database so it's accessible from any device
Build it locally first, I'll deploy when it's ready
Build this in the /Users/ryanvasquez/reading-tracker directory
